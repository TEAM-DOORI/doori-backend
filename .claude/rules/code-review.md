# Code Review Rules

리뷰할 때 아래 항목을 우선 확인한다.

## 확인 항목

- 요구사항을 제대로 만족하는가
- 예외 처리가 빠진 부분은 없는가
- 불필요한 중복 코드가 있는가
- 네이밍이 일관적인가
- 책임 분리가 적절한가
- 테스트가 가능한 구조인가
- PR 범위를 벗어난 변경이 섞여 있지 않은가
- 보안상 위험한 코드가 포함되어 있지 않은가
- 불필요한 로그 또는 디버깅 코드가 남아 있지 않은가

## 리뷰 방식

- 단순 스타일 지적보다 실제 문제 가능성을 우선한다.
- 수정이 필요한 이유를 함께 설명한다.
- 가능하면 개선 방향을 간단히 제시한다.
- 심각도 기준으로 분류한다.
