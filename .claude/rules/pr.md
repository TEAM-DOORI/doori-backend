# PR Rules

## PR 제목 규칙

- 형식: `[이모티콘 타입] 작업 내용 (#이슈번호)`
- 예: `[🔨 Fix] Notion DB 스키마 자동 매핑 처리 (#2)`

## PR 본문 필수 규칙

- 반드시 `Closes #이슈번호` 또는 `Fixes #이슈번호` 포함
- 템플릿 섹션을 유지해서 작성
  - 연관 이슈
  - 변경 요약
  - 구현 상세
  - 테스트 결과
  - 체크리스트

## 리뷰/품질 규칙

- PR 범위는 한 가지 목적에 집중한다.
- API/DB 영향이 있으면 본문에 명시한다.
- 실행한 검증 명령과 결과를 본문에 남긴다.
- CI 실패 상태에서는 머지하지 않는다.

## Notion 연동 규칙

- PR 본문의 이슈 번호(`Closes/Fixes #N`)는 동기화의 핵심 입력이다.
- 이슈 번호 누락 또는 `#0`은 동기화 실패 원인이 될 수 있다.
- 머지 후에도 이벤트 기반 워크플로우이므로, 필요한 이벤트가 실제로 발생했는지 확인한다.
