현재 변경사항을 PR 올리기 전에 사전 코드 리뷰해줘.

반드시 아래 규칙 파일을 우선 참고해:
- `.claude/rules/code-review.md`
- `.claude/rules/pr.md`

확인할 것:
- git status
- git diff
- 변경된 파일
- 예외 처리 누락 여부
- 중복 코드 여부
- 네이밍 일관성
- 책임 분리
- 테스트 가능 여부
- PR 범위를 벗어난 변경 여부
- 보안상 위험한 코드 포함 여부
- 워크플로우/자동화 안정성

## 반드시 수정 필요

-

## 개선하면 좋은 부분

-

## 문제 없어 보이는 부분

-

## 최종 판단

주의:
- 변경사항에 없는 내용을 상상하지 마.

- 수정이 필요한 경우 이유를 같이 설명해.
