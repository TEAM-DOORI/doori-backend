현재 변경사항을 커밋 가능한 상태로 정리해줘.

반드시 아래 규칙 파일을 우선 참고해:
- `.claude/rules/commit.md`
- `.claude/rules/pr.md`

작업 순서:
1. `git status --short` 로 변경 파일 확인
2. `git diff`로 변경 의도/범위 확인
3. 범위를 벗어난 파일이 있으면 커밋에서 제외
4. 필요한 검증 실행
   - YAML 변경 시: YAML 파싱 확인
   - JS 스크립트 변경 시: `node --check`
   - 백엔드 로직 변경 시: 관련 테스트 또는 `./gradlew test`
5. 규칙에 맞는 커밋 메시지 제안 후 커밋
